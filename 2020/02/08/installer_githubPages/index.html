Github Pages 博客搭建

Ruby + Jekyll + Github Pages

Posted by Law4D on February 8, 2020

Github Pages是Github免费提供的静态网站托管服务，不需要任何后端语言来支持。

  • 仓库空间不大于1G
  • 每个月的流量不超过100G
  • 每小时更新不超过 10 次

本文主要介绍如何使用Jekyll搭建个人Github Pages。

Jekyll是什么？

Jekyll 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过一个转换器（如 Markdown）和我们的 Liquid 渲染器转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。

选择Jekyll后，可以更加专注于博客内容本身，把网页排版交给Jekyll完成 :P

更新Ruby

首先查看ruby的版本，本文安装的jekyll 4.0.0要求ruby版本2.4.0以上。

1

$ ruby -v

如果版本低于2.4.0，需要先更新ruby版本。这里我们可以使用RVM进行ruby的版本管理和更新。

安装RVM

本文采用的是离线安装RVM，上Github下载RVM。

1
2
3

$ git clone https://github.com/rvm/rvm.git rvm
$ cd rvm/bin
$ ./rvm-installer

安装好之后在.bash_profile最后追加

export PATH=$PATH:/Users/xxx/.rvm/

其中xxx为当前用户名

1
2
3

$ cd ~/
$ vi .bash_profile
$ source ~/.bash_profile

到此RVM安装完成。

安装Jekyll

至此，可以开始安装Jekyll，如果在OSX平台上，可以直接输入

1

$ gem install jekyll

最后等着安装完成。最后编译博客的工具安装完成！🎉

Github Pages

去Github主页建立一个新的repository，命名为username.github.io，username为用户ID。

1
2
3
4
5
6
7

$ git clone https://github.com/law4D/law4D.github.io.git law4D.github.io
$ cd myblog
$ jekyll build -d ../law4D.github.io
$ cd ../law4D.github.io
$ git add --all
$ git commit -m "first commit"
$ git push -u origin master
